JSonSails

A framework for the creation of userdata-driven personalized web apps

Technology

  • Babel | transpile es6 to compatible javascript
  • Webpack | bundle assets for production
  • React.js | frontend library
  • jQuery | legacy ui components
  • Redux | functional app state
  • Django | backend
  • PostgreSQL | database
  • Sphinx | documentation generator
  • Selenium | browser automation for integration testing
  • Tape | js test framework

Purpose

JSonSails is a complete solution for creating and hosting user-data driven webapps. It creates a new abstraction of data over a set of users that simplifies webapp creation and administration.

The basic building block is the Node, composed of HTML, CSS, and JS. Code is written normally, with the addition of some JSonSails API functions. Nodes are used to layout the page, and are combined into Modules which can be standalone apps or combined and nested to create more complex and extensible applications.

JSonSails has an 'Easy' mode interface where code is not shown. Non-technical users can build an app from prebuilt modules via drag-and-drop, then customize each module through a graphical user interface.

Typical applications are surveys, progress reports, and dashboards.

I created the in-browser IDE and module rendering engine. I also developed the node.js backend that handles pushing data in real-time to browsers. I collaborated on the design of the database.

Documentation

I wrote the api docs, which are extracted from source, and helped to write the main documentation.

I developed a pipeline from jsdoc source comments using js2md to sphinx. It also pulls out code snippets into the documentation so that it is always up to date with code changes.
